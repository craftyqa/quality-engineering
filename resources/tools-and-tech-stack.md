# Tools & tech stack

Quality engineering sits on whatever your product uses. This file is for **your** stack and tools worth knowing—not an exhaustive industry list.

## Categories to map to your job

| Area | Examples (illustrative) | Your tools |
|------|-------------------------|------------|
| Test management | TestRail, Zephyr, Xray, spreadsheets | |
| Automation (UI) | Selenium, Playwright, Cypress, Appium | |
| Automation (API) | REST Assured, Postman + Newman, Karate | |
| Performance | k6, JMeter, Gatling | |
| CI/CD | GitHub Actions, GitLab CI, Jenkins, Azure DevOps | |
| Monitoring / logs | Datadog, Splunk, ELK, cloud-native tools | |
| Quality / delivery signals | DORA-style dashboards, incident tools, release analytics (often shared with SRE/EM) | |

## Learning approach

- Go **deep** on one automation tool and one API testing approach before chasing every new framework.
- Learn enough **CI** to own test jobs, artifacts, and failure signals.
- At **Staff / manager** scope, know how your org **measures** escapes, lead time, and stability—enough to argue for change without gaming metrics.
